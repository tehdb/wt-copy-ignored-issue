# wt copy-ignored issue — reproduction repo

Minimal reproduction for a bug in [`worktrunk`](https://github.com/max-sixty/worktrunk) where
`wt step copy-ignored` skips files that are re-ignored via a negation + re-ignore pattern in
`.gitignore`.

## Steps to reproduce

### 1. Clone this repo as bare

```bash
git clone --bare https://github.com/tehdb/wt-copy-ignored-issue.git .git
```

### 2. Create a worktree for `main`

```bash
git worktree add main main
cd main
```

### 3. Create the gitignored file in the primary worktree

```bash
cat > src/main/resources/logback-spring.xml << EOF
<?xml version="1.0" encoding="UTF-8"?>

<configuration>
    <appender name="CONSOLE"
              class="ch.qos.logback.core.ConsoleAppender">
        <layout class="ch.qos.logback.classic.PatternLayout">
            <Pattern>
                %highlight(%-5level) [%blue(%t)] %yellow(%C{1}): %msg%n%throwable
            </Pattern>
        </layout>
    </appender>
    <root level="info">
        <appender-ref ref="CONSOLE"/>
    </root>
</configuration>
EOF
```

Confirm git treats it as ignored:

```bash
git check-ignore -v src/main/resources/logback-spring.xml
# output: .gitignore:48:src/main/resources/logback-spring.xml	src/main/resources/logback-spring.xml
```

Review the relevant `.gitignore` rules

```gitignore
# generated by Spring Boot Initializr — un-ignores all source files
!**/src/main/**

# re-ignores the local config file
src/main/resources/logback-spring.xml
```

### 4. Create `.worktreeinclude`

```bash
cat > .worktreeinclude << EOF
.env
.venv/
build/
src/main/resources/logback-spring.xml
EOF
```

### 5. Create project config with `copy-ignored` hook

```bash
wt config create --project
cat > .config/wt.toml << EOF
[post-start]
copy-include = "cp {{ primary_worktree_path }}/.worktreeinclude {{ worktree_path }}/"
copy = "wt step copy-ignored"
EOF
```

### 6. Switch to a new feature worktree

```bash
wt switch -c some-feature
# if prompted about worktree-path: answer y
```

### 8. Verify

```bash
ls -la src/main/resources/
```

## Expected behavior

`logback-spring.xml` is present in `some-feature/src/main/resources/`, consistent with all other entries in `.worktreeinclude`.

## Actual behavior
`logback-spring.xml` is missing. All other entries in `.worktreeinclude` (`.env`, `.venv/`,
`build/`) are copied successfully.
